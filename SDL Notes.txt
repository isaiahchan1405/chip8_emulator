SDL_Window
 - A type defined by SDL that represents an operating system window
 - Implementation details are hidden, only needed to work with a pointer

SDL_Renderer
 - An object that knows how to draw graphics into a window
 - Sorta is like the paintbrush to the canvas(SDL_Window)

SDL_Init
 - Before SDL is used, its subsystems must be initialized
 - SDL_Init(..) is similiar to start_up_everything();

SDL_INIT_VIDEO
 - SDL contains several subsystems
   a. Video
   b. Audio
   c. Joystick
   d. Networking - Through extensions
 - SDL_INIT_VIDEO would only be initialized
 - 0 -> success
   non-zero -> failure

Error printing
 - fprintf(stderr,
        "SDL_Init failed: %s\n",
        SDL_GetError());
 - *fprintf prints to the specific output stream
 - stderr: standard error output(prints to terminal)
 - SDL_GetError(): Returns a string describing the last SDL error

SDL_RENDERER_ACCELERATED
 - Requests hardware acceleration, gets the GPU instead of the CPU to draw everything
 - It is usually faster
