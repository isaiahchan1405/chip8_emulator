Intro
    - The original implementation of the Chip-8 language has 36 different instructions
    - All instructions are 2 bytes long and are stored most-significat-byte first.
    - In memory, the first byte of each instruction should be located at an even address
    - If a program has sprite data, it should be padded so any instructions following it will be properly situated in RAM
    - Listings
      a. nnn / addr: A 12-bit value, the lowest 12 bits of the instruction
      b. n / nibble: A 4-bit value, the lowest 4 bits of the instruction
      c. x: A 4-bit value, the lower 4 bits of the high byte of the instruction
      d. y: A 4-bit value, the upper 4 bits of the low byte of the instruction
      e. kk / byte: An 8-bit value, lowest 8 bits of the instruction
    
Registers
    - 16 General 8-bit registers 
      a. 8 bit each, usually referred as Vx (x is a hexadecimal digit)
    - 1 16-bit register
      b. Called I
      c. Used to store memory addresses
      d. Only the lowest/rightmost 12 bits are usually used
    - VF register
      a. 8 bits, used to store results of logical or arithmetic operations
      b. Should not be used by any program, as it is used as a flag by some instructions
    - 2 special purpose 8-bit registers
      a. The delay and sound timers
      b. When they are non-zero, they are automatically decremented at a rate of 60Hz
    - Pseudo-registers
      a. Not accessible from Chip-8 programs
      b. Program Counters(PC) and Stack pointer(SP)
      c. Program Counters(PC) - should be 16 bit
         - Stores the currently executing address
      d. Stack Pointer(SP) - should be 8 bit
         - Point to the topmost level of the stack
    - The stack
      a. An array of 16 16-bit values
      b. Stores the address that the interpreter should to when finished with a subroutine
      c. Chip-8 allows for up to 16 levels of nested subroutines

Timers
    The Delay Timer
    - Active whenever the delay timer register(DT) is non-zero
    - Only subtracts 1 from the value of DT at a rate of Hz, when DT reaches 0, it deactivates

    The Sound Timer
    - Active whenever the sound timer register(ST) is non-zero
    - Decrements ay a rate of 60Hz, as long as ST's value is greater than zero, the Chip-8 buzzer will sound
    - Only deactivates when the ST reaches 0
    - Sound produced only has 1 tone, the frequency of the tone is decided by the author of the interpreter

The Keyboard
    - The computers which originally used the Chip-8 language had a 16-key hexadeximal keypad
    - This layout however, must be mapped into various other configurations to fit the modern keyboard platforms

The Display
    - The original implementation was a 64x32-pixel monochrom display

        --------------------------------
    - Chip-8 draws graphics on screen through the use of sprites
    - *sprites: A group of bytes which are a binary representation of the desired picture
    - Chip-8 sprites may be up to 15 bytes, having a possible sprite size of 8x15
